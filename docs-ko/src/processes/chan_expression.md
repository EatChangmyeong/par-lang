# 채널과 연결

이전 장에서는 `do`/`in`식을 사용해 값을 만들었다.

```par
def Message = do {
  let builder = String.Builder
  builder.add("Hello")
  builder.add(", ")
  builder.add("World")
} in builder.build
```

이 문법으로는 프로세스, 즉 명령의 나열을 실행하고 마지막에 식을 반환할 수 있다.

그런데 어떤 조건을 확인해서 **조기에 반환**하는 것은 가능할까? Par에는 *return* 키워드는 없지만, 더욱 심오한 메커니즘을 사용해 바로 이 동작뿐만 아니라 다른 동작도 구현할 수 있다.

`do`/`in` 문법 역시 더욱 근본적인 기능인 `chan`식과 연결이라는 명령을 합친 **문법 설탕**에 불과하다. 다음 코드는...

```par
do {
  <process>
} in <value>
```

다음 코드와 완전히 같다.

```par
chan result {
  <process>
  result <> <value>
}
```

이 코드가 무슨 의미인지 찬찬히 뜯어 보자.

## `chan`식

```par
chan inner {
  // 명령을 작성하는 곳
}
```

이 식은 새로운 채널을 만들고, 중괄호 안에 있는 명령을 실행하는 프로세스를 생성한다. 이 프로세스는 새로운 채널의 한쪽 끝을 사용할 수 있고, 여기서는 `inner`라는 이름을 사용한다.

`chan`식 자체는 이 채널의 반대쪽 끝으로 평가된다. 이 값이 식의 반환값이 된다.

**다음 내용이 핵심이다.** 식 자체의 타입이 `T`일 경우, 블록 안에서는 `inner`의 타입이 `dual T`가 된다.

다음 장에서 쌍대 타입을 자세히 배울 수 있지만, 지금은 **채널의 한쪽이 타입이라면, 반대쪽은 그 쌍대 타입이다**라는 직관을 가지는 것으로 충분하다. 한쪽에서 주면 반대쪽에서는 받고, 그 반대도 성립한다.

예를 들어, `chan`식 자체가 `Int`로 평가된다면, 그 반대쪽인 `dual Int`는 정수를 *요구*하는 것에 해당한다.

## `<>` 명령을 사용한 연결

위와 같은 요구를 만족하는 데는 연결 명령이 가장 쉽다. 타입 `T`의 값 `x`와 타입 `dual T`의 값 `y`가 있다면,

```par
x <> y
```

위의 명령으로 **두 채널의 끝점을 연결**한 뒤 프로세스를 종료한다.

예를 들어,
```par
def FortyTwo: Int = chan out {
  out <> 42
}
```

이 프로그램은 완전한 프로세스 문법을 사용해 정수 `42`를 반환한다.

다른 명령과 같이 `<>` 명령 역시 프로세스가 아니라 값(여기서는 두 채널)을 조작한다. 프로세스와 값을 연결하는 것이 아니라 두 채널을 서로 연결하는 것이다.

> **`chan` 안의 프로세스는 반드시 종료되어야 한다.** `chan` 안에 작성하는 모든 프로세스는 둘 중 하나의 명령으로 끝나야 한다.
> - **연결 명령** (`x <> y`)
> - **탈출 명령** (`x!`)은 다음 장에서 배운다.
> 
> 프로세스를 종료하는 방법은 위의 두 명령 이외에는 없다. 이 명령을 사용하지 않고 `chan` 블록의 끝에 도달한다면 컴파일 오류가 된다. 다소 이상한 조건으로 보일 수 있지만, 이렇게 해야 Par의 동시성 성질을 만족시킬 수 있다.
> 
> 한편 `do`/`in` 블록은 스스로 종료되는 대신 `in` 다음에 오는 식으로 진행한다.

이 명령을 사용해 **조기 반환**을 하는 함수를 작성해 보자.

이 함수는 자연수(`Nat`)을 전달받아 `String`을 반환한다.

- 0을 전달받으면 문자열 `"<nothing>"`을 반환한다.
- 그렇지 않으면 우물정자(`"#"`) `n`개로 이루어진 문자열을 반환한다.

구현하면 다음과 같다.

```par
module Main

import {
  @core/Nat
  @core/String
}

dec HashesOrNothing : [Nat] String
def HashesOrNothing = [n] chan result {
  let isZero = n == 0
  isZero.case {
    .true! => {
      result <> "<nothing>"
    }
    .false! => {}
  }

  let builder = String.Builder
  Nat.Repeat(n).begin.case {
    .end! => {}
    .step remaining => {
      builder.add("#")
      remaining.loop
    }
  }

  result <> builder.build
}
```

코드를 분석해 보자.

- `n == 0` (`iszero`)은 `Bool`을 반환하며, 이 값은 `.true!`와 `.false!` 분지가 있는 [분기](../types/either.md)이다.
- `.true!` 분지에서는 즉시 연결 명령을 수행하여 `"<nothing>"`을 반환한다. 이때 프로세스가 *종료*된다.
- `.false!` 분지에서는 일단 아무것도 하지 않는다. 실제 작업은 `.case`문 이후에 수행된다.
- `String.Builder`를 사용해 반환값을 생성하며, 이때 `Nat.Repeat`를 사용해 정확히 `n`회 반복한다.
- 마지막으로 빌더의 결과값을 `result`와 연결하여 프로세스를 종료한다.

다음 장에서는 쌍대성을 활용하여 반환뿐만 아니라 반대쪽 채널을 단계적으로 생성하는 법을 배워 보자!
