# 명령

프로세스는 `let`문과 명령을 나열한 것이다. 이전 장에서 [`let`문](./do_expression.md#let문)을 다루었으니 이제는 현실적인 상황을 다루는 작은 예제를 살펴보면서 실제 명령을 배워 보자.

그런데 **애초에 명령이 무엇인가?** 명령은 명령형 프로그래밍에서의 구문과 비슷하지만, 중요한 차이점이 있다. 명령형 프로그래밍의 구문 중에는 결과값을 얻는 것이 아니라 부작용(side-effect)을 일으킬 목적으로 평가되는 식인 경우가 많다. 한편 Par의 명령은 식과는 *전혀* 다르며, 문법상으로도 별개의 분류에 속한다.

모든 명령에는 그 명령의 **주어** 역할을 하는 채널이 있으며, 보통 지역 변수를 사용한다. 명령이 실행되고 나면 주어의 **타입이 변환된다**. 이것이 바로 명령형처럼 보이는 프로세스 문법에 엄밀하지만 편리한 세션 타입을 부여하는 단초가 되는, 명령형 언어의 구문과 프로세스 명령의 차이점이다.

**이제부터 직접 확인해 보자!**
